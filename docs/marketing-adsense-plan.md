# Google AdSense 導入計画

**作成日:** 2026-03-28
**担当:** マーケティング
**Sprint:** 3（タスク 3-5）

---

## 概要

こんにゃん堂にGoogle AdSenseを導入し、初期収益化の基盤を構築する。
ユーザー体験（占い没入感）を最優先としつつ、サイト運営コスト（API費用・ドメイン）を広告収益でカバーすることを目指す。

---

### 設計内容

#### 1. AdSense審査の前提条件

Googleが公開している審査要件を満たす必要がある。

| 要件 | こんにゃん堂の現状 | 対応 |
|------|-------------------|------|
| 独自ドメインでの運営 | konnyandou.jp（取得済み） | 対応済み |
| 十分なオリジナルコンテンツ | AI生成の占い結果は毎回ユニーク | 補強が必要（後述） |
| プライバシーポリシーの掲載 | 未設置 | 要作成 |
| 利用規約の掲載 | 未設置 | 要作成 |
| サイトの操作性・ナビゲーション | BottomTabNav実装済み、4ページ構成 | 概ね問題なし |
| 十分なページ数・テキスト量 | 4ページ（トップ/タロット/チャット/履歴） | やや少ない。静的コンテンツページの追加推奨 |
| Google Search Consoleへの登録 | 未登録 | 要登録 |
| ads.txt の設置 | 未設置 | 審査通過後に設置 |
| 禁止コンテンツに該当しないこと | 占いはエンタメ扱いで問題なし | 問題なし |

#### 2. 審査通過のための準備事項

**優先度P0（審査前に必須）**
- プライバシーポリシーページの作成（`/privacy`）
  - 個人情報の取扱い、Cookie利用、広告配信に関する記述
  - Google AdSense利用に関する記述（Googleが使用するCookie等）
- 利用規約ページの作成（`/terms`）
  - AI占いの免責事項（エンタメ目的、判断の根拠にしない旨）
  - サービス内容の説明
- Google Search Console への登録・サイトマップ送信
  - Sprint 3 タスク 3-2（sitemap.xml + robots.txt）と連携

**優先度P1（審査通過率を上げるために推奨）**
- 「こんにゃん堂について」ページの作成（`/about`）
  - サービスの説明、ノアとツキのキャラクター紹介
  - 運営者情報（サイト名・連絡先）
- お問い合わせ手段の設置
  - 最低限メールアドレスの記載、またはGoogleフォームへのリンク
- 静的コンテンツの充実（Sprint 4のSEOタスクと連携）
  - 星座別の解説ページ（12ページ）
  - タロットカード解説ページ（22枚分）
  - これらはSEO施策とも兼ねるため、Sprint 4と合わせて進めるのが効率的

**優先度P2（申請前に確認）**
- 全ページのmeta description設定（Sprint 3 タスク 3-3）
- OGP画像の設定完了確認（Sprint 1 タスク 1-4）
- サイト表示速度の確認（Vercel + Next.jsなので概ね問題ないはず）

#### 3. 広告配置の推奨位置

こんにゃん堂のデザインはmax-width 430pxのモバイルファーストレイアウトのため、
広告もモバイル前提で設計する。

**配置方針: 占い体験を中断しない位置に限定**

| 配置位置 | 広告タイプ | 理由 |
|---------|-----------|------|
| 占い結果の下部（シェアボタンの下） | ディスプレイ広告（レスポンシブ） | 占い結果を読み終えた後なので体験を損なわない。最もCTRが期待できる |
| 履歴ページ（一覧の間） | インフィード広告 | 履歴閲覧は情報探索モード。広告への抵抗感が低い |
| ページ遷移時 | アンカー広告（自動広告） | Google側が最適タイミングで表示。ユーザーが閉じられる |

**配置しない場所（禁止）**
- 占いの入力フォーム周辺（星座選択、チャット入力）
- 占い結果のテキスト途中（ノアの鑑定文の中）
- BottomTabNav周辺（誤タップ誘発、ポリシー違反リスク）
- ファーストビュー全体を覆う形（インタースティシャルの乱用）

#### 4. 実装方法の概要（Next.js + Vercel）

**Step 1: AdSenseアカウント取得・審査申請**
- Google AdSenseアカウント作成
- サイトURLとして `https://konnyandou.jp` を登録
- 審査用コードスニペットを `<head>` に追加

**Step 2: 審査コードの実装**
- `src/app/layout.tsx` の `<head>` にAdSense審査用 `<script>` タグを追加
- Next.js の `<Script>` コンポーネント（`strategy="afterInteractive"`）を使用
- 環境変数 `NEXT_PUBLIC_ADSENSE_CLIENT_ID` で管理

**Step 3: 審査通過後の広告ユニット実装**
- AdSense広告コンポーネントを作成（`src/components/AdUnit.tsx`）
  - `'use client'` が必要（クライアントサイドで広告スクリプトを実行）
  - `useEffect` でAdSenseスクリプトの初期化
  - 広告スロットIDをpropsで受け取る汎用コンポーネントにする
- 各ページの適切な位置に `<AdUnit>` を配置

**Step 4: ads.txt の設置**
- `public/ads.txt` にGoogleが指定する内容を記載
- Vercelデプロイ時に自動的に `/ads.txt` として配信される

**Step 5: パフォーマンス対策**
- 広告スクリプトは `strategy="lazyOnload"` で遅延読み込み
- 広告枠にはCLS対策として固定サイズの `min-height` を設定
- Core Web Vitalsへの影響を監視（Vercel Analytics）

#### 5. 想定収益のざっくり試算

**前提条件**

| 項目 | 値 | 根拠 |
|------|-----|------|
| 月間PV（初月） | 3,000 PV | 新規サイト、SNS運用初期。X投稿+自然検索でDAU 100程度 |
| 月間PV（3ヶ月後） | 10,000 PV | SEO効果+SNS成長+リピーター定着 |
| 月間PV（6ヶ月後） | 30,000 PV | SEOコンテンツ充実+相性占いバイラル |
| 広告表示率 | 70% | 一部ページで非表示+AdBlock利用者 |
| ページあたり広告数 | 1.5枚 | 結果ページ2枚、他1枚程度 |
| ページRPM | 100-200円 | 占い・エンタメジャンルの日本語サイト相場 |

**収益予測**

| 時期 | 月間PV | 月間収益（概算） | 備考 |
|------|--------|----------------|------|
| 初月 | 3,000 | 300-600円 | サイト運営コストの一部カバー |
| 3ヶ月後 | 10,000 | 1,000-2,000円 | ドメイン費用をカバー |
| 6ヶ月後 | 30,000 | 3,000-6,000円 | API費用+ドメイン費用をカバー |
| 1年後 | 100,000 | 10,000-20,000円 | 目標値。Pro課金との組み合わせで黒字化 |

**注意:** AdSense単体での大きな収益は期待しない。初期の収益化手段として位置づけ、
将来的にはPro版課金（Stripe）やアフィリエイトとの組み合わせで収益を拡大する。
