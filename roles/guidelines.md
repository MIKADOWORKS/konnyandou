# 全エージェント共通ガイドライン

このファイルは全ロール共通の行動規範です。
起動時に自分のロールファイルと合わせて必ず読んでください。

---

## 1. 起動時の手順

1. `CLAUDE.md` を読む（プロジェクト概要・チーム体制）
2. `roles/guidelines.md`（このファイル）を読む
3. 自分のロールファイル `roles/xxx.md` を読む
4. タスクの実行に入る

## 2. コミュニケーション原則

### 報告は「結論ファースト」
- 最初の1-2行で結果を述べる（成功/失敗/ブロッカーあり）
- 詳細は箇条書きまたはテーブルで簡潔に
- 長文の散文は禁止。構造化されたフォーマットで書く

### 不明点は推測しない
- 仕様が曖昧な場合は「前提: 〇〇と解釈して進めます」と明記する
- 判断に迷う選択肢が2つ以上ある場合は、選択肢を列挙してディレクターに判断を仰ぐ

### コンテキスト消費を意識する
- 報告は必要最小限の情報量で。冗長な説明は不要
- ファイルの全文引用は避け、変更箇所の要約を書く
- 大量のコード出力よりも「何を変えたか」の説明を優先

## 3. コード変更のルール

### 変更前
- 変更対象のファイルを必ず先に読む
- Next.js 16 を使用中。コードを書く前に `node_modules/next/dist/docs/` を確認
- 他のエージェントが同時に変更しそうなファイルがあれば、ディレクターに報告

### 変更中
- TypeScript strict mode を維持
- `'use client'` は必要最小限（Server Component 優先）
- Tailwind CSS v4 でスタイリング（CSS-in-JS 禁止）
- 入力バリデーションには Zod を使用
- Anthropic API 呼び出しは `src/lib/claude.ts` の `callClaude()` 経由
- システムプロンプトは `src/lib/prompts.ts` に集約（他ファイルでの直接定義禁止）
- URL は `src/lib/constants.ts` の `SITE_URL` を参照（ハードコード禁止）

### 変更後
- `npm run build` で必ずビルド確認
- コミットメッセージは日本語OK。プレフィックス: `feat:` / `fix:` / `refactor:` / `docs:`
- 変更内容を簡潔に報告（変更ファイル一覧 + 主な変更点）

## 4. ファイル所有権

同一ファイルを複数エージェントが同時に変更するのを防ぐため、以下の所有権を守る。

| ファイル/ディレクトリ | 主担当 | 備考 |
|----------------------|--------|------|
| `src/lib/claude.ts` | エンジニア | API共通クライアント |
| `src/lib/prompts.ts` | エンジニア | ノアのキャラ設定変更時はディレクター承認 |
| `src/lib/rate-limit.ts` | エンジニア | |
| `src/lib/tarot-data.ts` | リサーチャー（データ）/ エンジニア（型） | |
| `src/lib/zodiac-data.ts` | リサーチャー（データ）/ エンジニア（型） | |
| `src/app/api/*/route.ts` | エンジニア | 新規APIはプロトエンジニアが初稿 |
| `src/app/*/page.tsx` | エンジニア / プロトエンジニア | 新ページはプロトが作成→エンジニアがレビュー |
| `src/components/*.tsx` | エンジニア | |
| `src/app/globals.css` | アートディレクター（デザイントークン）/ エンジニア（実装） | |
| `public/images/` | アートディレクター | |
| `docs/` | 全ロール（自分の担当領域のドキュメント） | |
| `roles/*.md` | ディレクター | |

## 5. 品質チェックリスト

コード変更を含むタスク完了時、以下を確認すること。

- [ ] `npm run build` が成功する
- [ ] TypeScript のエラーがない
- [ ] ハードコードされたURL・APIキーがない
- [ ] `'use client'` の不要な追加がない
- [ ] 新規ファイルが適切なディレクトリに配置されている
- [ ] 既存のテスト（将来追加時）が壊れていない

## 6. 禁止事項

- **本番環境への直接操作**（Vercel設定変更、DNS変更等）→ オーナーに依頼
- **環境変数の値をログやレポートに含める**
- **他ロールの所有ファイルを無断で変更する**
- **`package.json` への依存追加を事前報告なしに行う**（ディレクターに報告後に追加）
- **破壊的な git 操作**（force push, reset --hard, branch -D）
